# Decisões do Projeto (v0)

## Por que monorepo?
Para manter `web`, `api` e `contracts` sincronizados (ABIs, tipos, scripts).

## O que vai para on-chain
- Registro mínimo do projeto (id, owner, referência/hash de metadados)
- Eventos imutáveis (criação, validação, marcos)

## O que vai para off-chain
- Descrição completa
- Métricas e relatórios
- Dados sensíveis

## Stack (congelada por enquanto)
- Web: Next.js + React
- Contracts: Solidity + Hardhat
- API: Node
- DB: Postgres
